# Parker 0.1.23 Release Notes

Status: Released

## Added

- Added reader bookmarks, so you can save important pages in a comic without changing your normal reading progress.
- Added bookmark detours, letting you jump back to a saved page for reference and then return to where you were reading.
- Added saved bookmarks to comic detail pages for quick access outside the reader.
- Added Library Timeline pages for character and team tags, making it easier to explore a character or team's reading history across your library.
- Added an optional setting to open single-volume series directly on their volume page.
- Added a dashboard prompt for Parker Social Insights participation when anonymous sharing is disabled.

## Changed

- Improved reader navigation for story arcs so previous and next stay within the actual arc metadata.
- Improved reader close behavior so Parker returns you to the page you launched the reader from more reliably.
- Improved bookmark management with searchable bookmark lists, compact rows, and inline renaming.
- Improved comic detail actions with clearer `Read`, `Continue`, and `Read Again` wording.
- Improved stack awareness on comic detail pages so existing stack membership is visible before adding.
- Character and team chips now open Library Timeline first, with Advanced Search still available for broader searching.
- Parker Social Insights now uses clearer naming and copy while preserving existing user preferences.

## Fixed

- Fixed bookmark revisit flows so checking an earlier page no longer accidentally disturbs your true resume position.
- Fixed story arc reader navigation so unrelated filler issues are not pulled into arc reading.
- Fixed reader exits from dashboard and other launch points that previously returned to comic detail instead of the original page.
- Fixed comic detail stack actions that could hide existing membership and lead to duplicate-add attempts.
- Fixed completed comics showing a generic `Read` action instead of `Read Again`.
- Fixed stack detail metadata so pencillers, characters, teams, and locations are shown consistently.
- Fixed comic detail tag sorting so metadata chips appear alphabetically.
- Fixed active sessions so normal browsing can refresh short-lived access cookies without unnecessarily sending users back to login.

## Validation

- Verified bookmark, reader, comic detail, timeline, settings, authentication, Social Insights, and migration coverage with focused API and browser tests.
- Verified Alembic reports a single head and upgrades cleanly against in-memory SQLite.
