# Parker 0.1.21 Release Notes

Status: In Progress

This file tracks the work landing in `0.1.21` so release notes can be built incrementally instead of reconstructed at release time.

## Changed

- Started modularizing the reader by splitting `app/templates/reader.html` into a thinner shared shell with extracted reader partials and a dedicated `static/js/reader.js` module, creating cleaner seams for future alternate reader modes such as archived vertical-scroll support.
- Added an initial browser-test foundation using Playwright against a live local app instance, plus the first reader smoke tests so frontend behavior can be exercised beyond API-only coverage.
- Expanded the Playwright smoke coverage to exercise real reader completion, advanced-search result loading, and series detail filtering flows against a live app instance.
- Extended the Playwright smoke suite to cover `Continue Reading`, reading-list-to-reader navigation, the cover browser, and navbar quick search using a richer seeded browser-test dataset.
- Added a second browser-test wave around reader keyboard contracts, incognito progress behavior, search handoff hydration, and continue-reading mutation flows so the highest-risk client interactions now have live end-to-end coverage.
- Added browser coverage for empty and error states plus a narrow-viewport mobile smoke pass, giving the suite checks around graceful no-data UX and the main responsive reading/navigation flows.
- Added accessibility-leaning keyboard/focus browser coverage plus saved-search and pull-list CRUD flows, and tightened quick-search `Escape` handling so keyboard dismissal now closes the results panel reliably.
- Added another browser-test pass for login/logout session lifecycle, saved-search reapply, pull-list item management from comic detail through list editing, and persisted reader preferences, while tightening logout cleanup so client-side session state is fully cleared.
