# Parker 0.1.21 Release Notes

Status: In Progress

This file tracks the work landing in `0.1.21` so release notes can be built incrementally instead of reconstructed at release time.

## Added

- Added an initial Playwright browser-test foundation against a live local app instance and expanded it into broad smoke coverage for the main client-side flows, including reader navigation, progress/completion, advanced search, series filtering, `Continue Reading`, cover browsing, quick search, login/logout, saved searches, pull lists, empty/error states, and narrow-viewport/mobile behavior.
- Added the first archived vertical-scroll reader slice behind a manual `Long View` toggle, with a separate stacked-page rendering path, page-index progress tracking from scroll position, and browser coverage proving the new mode coexists with the existing paged reader.
- Added sparse per-book reader overrides in local storage for content-specific behavior, allowing settings like `Long View`, `Double Page`, and Manga/RTL mode to persist for one comic without leaking into unrelated books while still respecting global reader defaults.

## Changed

- Modularized the reader by splitting `app/templates/reader.html` into a thinner shared shell with extracted reader partials and a dedicated `static/js/reader.js` module, creating cleaner seams for alternate reader modes and future reader work.
- Replaced the temporary frontend route-map workaround with FastAPI's supported `iter_route_contexts()` traversal, reducing custom router-flattening code in `app/core/utils.py` while keeping nested router prefixes and tags resolved correctly for frontend route injection.

## Fixed

- Fixed quick-search keyboard dismissal so pressing `Escape` now closes the results panel reliably during keyboard-driven navigation.
- Fixed logout cleanup so client-side session state is fully cleared when ending a session.
- Fixed incognito reader behavior so per-book reader overrides remain ephemeral for the current session instead of being written back to local storage.

## Validation

- `.\.venv\Scripts\python.exe -m pytest tests/browser -m browser`
