# Parker 0.1.21 Release Notes

Status: Ready for Release

`0.1.21` is a reader-focused release that lays the groundwork for alternate reading modes, adds the first `Long View` experience for web-comic style archives, and strengthens confidence in the client with broader real-browser coverage.

## Added

- Added the first `Long View` reader mode for tall, web-comic style archives, including a stacked-page rendering path and scroll-based progress tracking that coexists with the traditional paged reader.
- Added per-book reader overrides for settings that commonly vary by title, allowing options like `Long View`, `Double Page`, and Manga/RTL mode to persist for one comic without leaking into unrelated books.
- Added an initial Playwright browser-test foundation and expanded it into broad real-browser coverage for the main client-side flows, including reader navigation, progress/completion, advanced search, series filtering, `Continue Reading`, cover browsing, quick search, login/logout, saved searches, stacks, empty/error states, and narrow-viewport/mobile behavior.

## Changed

- Modularized the reader into a thinner shared shell with extracted partials and a dedicated `static/js/reader.js` module, creating cleaner seams for alternate reader modes and future reader work.
- Replaced the temporary frontend route-map workaround with FastAPI's supported `iter_route_contexts()` traversal, reducing custom router-flattening code in `app/core/utils.py` while keeping nested router prefixes and tags resolved correctly for frontend route injection.
- Rebranded the user-facing `Pull Lists` feature as `Stacks` across the UI, reader context labels, search surfaces, and supporting docs to better reflect that the feature is a manual, curated reading bucket rather than a future-issue subscription system, while intentionally keeping the current internal API, model, and database names stable for now.
- Renamed the user-facing page URLs for stacks from `/pull-lists` to `/stacks` and left compatibility redirects in place so existing bookmarks and links keep working.

## Fixed

- Fixed long-view arrow navigation so stale scroll-position restore work can no longer override a newer page jump and snap the reader back to the previous archive page.
- Fixed quick-search keyboard dismissal so pressing `Escape` now closes the results panel reliably during keyboard-driven navigation.
- Fixed logout cleanup so client-side session state is fully cleared when ending a session.
- Fixed incognito reader behavior so per-book reader overrides remain ephemeral for the current session instead of being written back to local storage.
